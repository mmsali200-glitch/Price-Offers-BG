### 3.4 طريقة عرض أسعار الموديولات ← v3
| الوضع | ما يظهر في العرض |
|-------|----------------|
| `إجمالي موحد` | أسماء الموديولات + سعر مجمّع واحد |
| `سعر كل موديول` | كل موديول بسطر وسعر مستقل |
| `إخفاء الأسعار` | أسماء الموديولات بدون أرقام |

**بند منفصل لكل موديول ← v3:**
```
كل موديول محدد يمكن تعيينه:
  □ مجمّع → يندمج مع باقي الموديولات في بند واحد
  ✓ بند منفصل → يظهر كسطر مستقل بسعره في جدول التسعير
```

**خصم على كل بند ← v3:**
```
كل موديول لديه حقل خصم %
السعر النهائي = السعر الأصلي × (1 - الخصم%)
```

## 5. Configurator التفاعلي — JavaScript

```javascript
var BASE = {{DEV_TOTAL}};  // إجمالي الموديولات الأساسية (بعد الخصومات)
var opts = {};             // الخيارات الاختيارية
var licCost = {{LIC}};     // تكلفة الترخيص
var supCost = {{SUP}};     // تكلفة الدعم

function fmt(n){ return String(n).replace(/\B(?=(\d{3})+(?!\d))/g, ','); }

function updateTotal(){
  var extra = 0;
  for(var k in opts){ extra += opts[k] || 0; }
  var dev = BASE + extra;
  var monthly = Math.round(dev / {{MONTHS}});
  
  // تحديث العناصر المرئية
  document.getElementById('total-dev').innerHTML  = fmt(dev) + ' {{CUR}}';
  document.getElementById('total-lic').innerHTML  = fmt(licCost) + ' {{CUR}}';
  document.getElementById('total-sup').innerHTML  = supCost ? fmt(supCost) + ' {{CUR}}' : '—';
  document.getElementById('total-inst').innerHTML = fmt(monthly) + ' {{CUR}}';
}
```

```
[سطر التعليمات — عربي أو إنجليزي حسب الاختيار]

═══ 📌 بيانات العميل ═══
اسم العميل / رقم العرض / التاريخ / العملة / الإصدار / القطاع

[لغة العرض: عربي/English]

═══ 📦 الموديولات ═══
[طريقة العرض: إجمالي/تفصيلي/مخفي]
[خصم إجمالي % إن وجد]
[سعر الصرف إن كان ≠ 1]
[قائمة المنفصلة بأسعارها]
[قائمة المجمّعة + إجمالي]

═══ ⭐ تطبيقات BG ═══
[كل تطبيق: تكلفة التطبيق + شهري]

═══ ➕ المكونات الاختيارية ═══

═══ 📅 مراحل التنفيذ ═══
[كل مرحلة: الاسم + المدة + المخرجات]

═══ 💳 التسعير والدفع ═══
[قيمة التطوير + الخصومات + تاريخ البدء + الأقساط]

═══ 🔑 الترخيص ═══
[النوع + الخادم + المستخدمون + التكلفة الشهرية]
[⚠ إرشادي — يُؤكَّد بعرض Odoo]
[تكلفة Odoo للفترة إن كانت مفعّلة]

═══ 🛟 الدعم الفني ═══
[المجاني + المدفوع + الباقة + المميزات]

═══ 📋 متطلبات التشغيل ═══
[لكل موديول: قائمة المتطلبات]

═══ 📝 وصف المشروع ═══
═══ 🔄 دورات العمل ═══
═══ 💬 ملاحظات الاجتماع ═══
═══ ✅ الملخص المالي ═══
[تطوير + ترخيص + دعم + الإجمالي الكلي إن فُعّل]
```

## 9. قواعد توليد إجمالي العرض الكلي

```
السعر النهائي للعرض يُحسَّب بهذا الترتيب:

1. مجموع الموديولات الأساسية (بعد خصم كل بند)
2. + تكلفة تطبيقات BG (مرة واحدة)
3. × (1 - خصم إجمالي %)
4. [اختياري] + تكلفة Odoo × عدد الأشهر × سعر الصرف

ملاحظات:
- تكلفة الترخيص الشهرية تُعرض منفصلةً دائماً كـ "متكررة"
- تكلفة BG الشهرية تُعرض منفصلةً "شهرياً"
- الإجمالي الكلي = تطوير + Odoo (إن اختير)
```
